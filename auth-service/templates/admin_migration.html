⚠️ Внимание

Операции миграции могут повлиять на работу системы. Рекомендуется создать резервную копию базы данных перед выполнением операций.

Что включает в себя миграция ADR-001:

  • Обновление схемы сервисов: Переход от простого массива разрешений к структурированным PermissionDef
  • Добавление временных меток: Поля createdAt и updatedAt для ролей
  • Создание стандартных сервисов: referal, calculators, system
  • Поддержка мягкого удаления: Разрешения помечаются как удаленные вместо физического удаления
  • Новые заголовки авторизации: X-User-Service-Roles и X-User-Service-Permissions

Схема новой авторизации

Service { Key: string // Уникальный ключ сервиса Name: string // Отображаемое название Description: string // Описание сервиса AvailablePermissions: []PermissionDef { Name: string // Идентификатор разрешения DisplayName: string // Отображаемое название Description: string // Описание разрешения DeletedAt: time.Time // Мягкое удаление } Status: string // active, deprecated, disabled CreatedAt: time.Time UpdatedAt: time.Time } Role { ServiceKey: string // Ссылка на Service.Key Name: string // Название роли Permissions: []string // Массив названий разрешений CreatedAt: time.Time UpdatedAt: time.Time }
