# Notification Service Configuration

## Управление настройками

Все настройки сервиса уведомлений теперь управляются через веб-интерфейс:

**URL:** `http://localhost/notification-settings`

**Доступ:** Только для системных администраторов

## Настройки SMTP

Следующие параметры настраиваются через веб-интерфейс:

- **SMTP Сервер** - адрес SMTP сервера (например: smtp.gmail.com, 172.16.0.201)
- **SMTP Порт** - порт SMTP сервера (обычно 587 для TLS, 465 для SSL)
- **Пользователь SMTP** - имя пользователя для аутентификации
- **Пароль SMTP** - пароль или токен приложения
- **Отправитель (From)** - email адрес отправителя
- **Использовать TLS** - включение шифрования TLS
- **Требуется аутентификация** - включение аутентификации SMTP
- **Метод аутентификации** - PLAIN, LOGIN, CRAM-MD5

## Настройки обработки

Параметры производительности и повторных попыток:

- **Максимум попыток** - количество попыток отправки при ошибке (1-10)
- **Размер батча** - количество писем в одном батче (1-100)
- **Задержка между батчами** - пауза между отправкой батчей в миллисекундах

## Тестирование

На странице настроек есть функция отправки тестового письма для проверки корректности конфигурации.

## Fallback настройки

В случае недоступности notification-service, auth-service использует fallback настройки из файла `.env`. Эти настройки должны быть синхронизированы с основными настройками.

## API

Для программного управления настройками доступно API:

- `GET /api/v1/config` - получение текущих настроек
- `POST /api/v1/config` - обновление настроек
- `POST /api/v1/notifications` - отправка уведомления

## База данных

Настройки хранятся в таблице `notification_configs` в базе данных PostgreSQL notification-service.
